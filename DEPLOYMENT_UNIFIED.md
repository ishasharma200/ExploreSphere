# Single-Deployment Setup Guide

Explore Sphere is now configured for **unified single-deployment** where the backend server serves both:
- REST API routes (`/api/*`)
- Frontend static files (React SPA)

## How it works

1. Frontend is built to `frontend/dist`
2. Backend middleware serves `frontend/dist` as static files
3. Backend has a catch-all route for SPA routing
4. Single deployment to Railway/Render

## Local testing

```bash
# Build frontend
cd frontend
npm run build

# Start backend (serves everything)
cd ../backend
npm start

# Access app at http://localhost:5000
```

## Deployment to Railway/Render

### Railway steps:
1. Create new project
2. Connect your GitHub repo
3. Set project root to `backend`
4. Environment variables:
   - `MONGO_URI=<your-mongo-uri>`
   - `JWT_SECRET=<strong-secret>`
   - `NODE_ENV=production`
   - `CLIENT_URL=https://<railway-generated-url>`

### Render steps:
1. Create new Web Service
2. Connect repo, set root to `backend`
3. Build command: (leave blank)
4. Start command: `node server.js`
5. Set same environment variables as Railway
6. `Procfile` is included for compatibility

## Files modified

- `backend/server.js` — Added static file serving + SPA catch-all
- `backend/package.json` — Added build script for frontend
- `frontend/dist/` — Built frontend files (generated by `npm run build`)

## Result

- Single URL for entire app
- No CORS issues between frontend and backend
- Easy one-click deployment
- Backend automatically builds frontend during `postinstall` hook
